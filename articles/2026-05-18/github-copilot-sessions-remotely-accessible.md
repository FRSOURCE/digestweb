## Summary & Key Takeaways

- GitHub Copilot sessions now support remote control.
- Developers can start work in VS Code or the CLI and continue on GitHub.com or GitHub Mobile.
- This feature aims to provide seamless continuity for Copilot-assisted development across different devices.
- The remote control functionality is now generally available.

## Our Commentary

This is a neat quality-of-life improvement for Copilot users. The idea of kicking off a coding session on my desktop and then reviewing or making quick edits with AI assistance from my phone is pretty compelling, especially for those moments when you're away from your main setup but a thought strikes. It speaks to the increasing integration of AI tools into every part of the development workflow, making them more ubiquitous and less tied to a single environment.
